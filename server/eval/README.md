# dflash quality evaluation

Tools for measuring whether dflash's pflash compression, prefix cache, and BSA
optimisations change the model's output vs a baseline (no caches, no
compression). The question they answer is "does flag X cause the model to
produce different bytes for the same prompt?", not "is the answer correct".

## quick A/B (`scripts/quality_ab_simple.py`)

Runs ~7 short conversational prompts against several server configs. For each
config it spawns a fresh `server/scripts/server.py`, fires the prompts in
sequence, then tears the server down. At the end it prints a markdown table
comparing each config against the matching baseline (configs ending in `_f16`
are compared against `baseline_f16` so attention precision is held constant).

```
PFLASH_TARGET=/path/to/target.gguf \
PFLASH_DRAFT=/path/to/draft-dir-or-safetensors \
PFLASH_BIN=server/build/test_dflash \
PFLASH_DRAFTER=/path/to/Qwen3-0.6B-BF16.gguf \
python3 server/scripts/quality_ab_simple.py
```

Configs are defined in `CONFIGS` near the top of the script. Each spawns one
server, then closes it. Wallclock for the default 8-config matrix on a single
RTX 3090: ~12 minutes including model-load time.

The harness includes a `baseline_2` config — identical to `baseline` — used as
a determinism sanity check. If `baseline` and `baseline_2` produce different
output for some prompt, that prompt is intrinsically nondeterministic on the
server and should be excluded from any cache-quality conclusion. Without this
check, cache-induced divergence and intrinsic noise are indistinguishable.

## MT-Bench data (`mt_bench/`)

Vendored from `lm-sys/FastChat`'s MT-Bench (Apache-2.0). 80 multi-turn
conversations across 8 categories. `SOURCE.txt` documents the exact upstream
URL.

## Limitations

This is a **sameness** test, not a capability test. It cannot tell you whether
the answer is correct, only whether it changed. For functional quality use a
graded benchmark (HumanEval+, LiveCodeBench, MMLU). Sample size of ~7 prompts
is small; a single divergence in a long output can move the whole exact-match
rate dramatically.

## Security Note

`quality_humaneval_plus.py` executes model-generated code via `exec()` in a
subprocess with a SIGALRM timeout.  This is NOT a security sandbox — generated
code has full access to the host filesystem, network, and process tree.  Only
run this harness on trusted models in an environment where arbitrary code
execution is acceptable (e.g., your local dev machine, a disposable VM).
